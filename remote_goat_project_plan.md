# Remote Goat Ecosystem - Master Build Plan

## Project Overview
Multi-agent business ecosystem combining:
1. **Remote Goat** - Agentic hustler platform (digidrifters.com)
2. **Tech Runner** - Technical setup service (sensorush.com)
3. **Applesamosa** - Business idea incubator (applesamosa.com)
4. **Multi-Agent System** - Femi + Loveable Rogue + Maulti interaction

## Core Philosophy
Based on Marcel Broschk's agentic company principles:
- Intent-based systems (not rule-based)
- Semi-autonomous execution (bounded processes)
- Multi-agent orchestration (specialized agents)
- Self-improving algorithms (learn from users)
- Progressive trust (narrow start, expand as earned)

## Technology Stack

### Remote Goat (Primary)
- **Domain:** digidrifters.com
- **Framework:** Next.js (agentic-ready)
- **Hosting:** Vercel (start free, scale as grows)
- **Payments:** Gumroad (quick launch, low complexity)
- **Database:** Supabase (real-time features)
- **AI Integration:** OpenAI API
- **Frontend:** Tailwind CSS (fast development)

### Tech Runner (Supporting)
- **Domain:** sensorush.com
- **Framework:** Astro (static speed)
- **Documentation:** Self-hosted n8n examples
- **Payments:** Stripe (B2B service needs control)
- **Booking:** Calendly (start free)
- **Contact:** Formspree (simple forms)

### Applesamosa (Incubator)
- **Domain:** applesamosa.com (existing, perfect fit!)
- **Framework:** Astro (static performance)
- **Content:** Multi-agent demos, blueprints
- **Integration:** Git-connected repo
- **Features:** Community voting, idea tracking

## Key Insights from Marcel Broschk Article

### What We're Applying:
1. **Intent-based systems** > rule-based automation
2. **Semi-autonomous agents** > simple chatbots
3. **Multi-agent orchestration** > single tools
4. **Self-improving algorithms** > static content
5. **Progressive trust** > reckless autonomy
6. **Grounded execution** > generic AI responses

### What This Means for Remote Goat:
- Systems learn from users (not just provide information)
- Agents understand intent (not just follow rules)
- Bounded autonomy (human oversight for strategy)
- Connects to real data/tools (not just text output)
